I focus on the engineering layer around LLM apps: runtime orchestration, workflow execution, evaluation loops, deployment tooling, and product-grade AI systems.

## Focus

| Area | What I care about |
| --- | --- |
| Agent Runtime | Explicit state, lifecycle control, deterministic task flow, traceable execution |
| Workflow / Skill Execution | Turning repeatable work into reliable skills, tools, and async workflows |
| Setup and Deployment | Low-friction local setup, desktop packaging, and deployable app scaffolds |
| Evaluation | Feedback loops, quality gates, badcase analysis, and measurable iteration |
| Product Systems | Desktop AI products, vertical agents, onboarding flows, and memory systems |

## Engineering Bias

- Make state explicit before making behavior clever.
- Prefer workflow reliability where a task is repeatable.
- Keep evaluation close to real product behavior.
- Build tools that shorten the path from prototype to deployable system.
- Treat runtime traces, badcases, and user-facing failures as first-class design inputs.
